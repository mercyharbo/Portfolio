## Overview

Briefly summarize the project in 2-3 paragraphs.

Include:

- What the product is.
- Who it serves.
- What you were responsible for.
- The most important result or outcome.

## Background

Explain the context that made the project necessary.

Prompts:

- What problem existed before this project?
- What business or user need drove the work?
- What constraints shaped the solution?
- What made the project technically interesting?

## Problem

Describe the main problems clearly.

Include:

- User pain points.
- Business pain points.
- Technical limitations.
- Operational bottlenecks.
- Performance, conversion, or maintainability issues.

## Goals

List the project goals.

Example:

- Improve checkout completion.
- Build a scalable frontend architecture.
- Reduce manual workflow steps.
- Improve dashboard usability.
- Improve Core Web Vitals.
- Create a premium brand experience.

## Research

Explain what informed the solution.

Include:

- User workflow review.
- Competitor or product references.
- Analytics or performance observations.
- Stakeholder requirements.
- Technical feasibility checks.

## Solution

Explain the implemented solution at a high level.

Include:

- Main product experience.
- Key user flows.
- Core frontend decisions.
- Important integrations.
- UI and interaction decisions.
- How the solution maps back to the original problem.

## Challenges

Include:

- The challenge.
- Why it was difficult.
- Options considered.
- Final decision.

## Performance

Describe performance work.

Include:

- Caching.
- Rendering strategy.
- Core Web Vitals improvements.

## Accessibility

Describe accessibility decisions.

Include:

- Semantic HTML.
- Keyboard support.
- Dialog behavior.
- Form labels and errors.
- Color contrast.
- Responsive text and layout.

## Results

Include measurable outcomes where possible.

Examples:

- Revenue generated.
- Conversion improvement.
- Time saved.
- Performance improvement.
- User growth.
- Workflow reduction.
- Stakeholder feedback.

If exact metrics are not available, use honest qualitative results.
